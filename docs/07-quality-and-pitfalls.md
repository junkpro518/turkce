# 07 — الجودة والمزالق

> هذه الوثيقة تحوّل دروس النسخ الثلاث الفاشلة إلى **حواجز صريحة**.

## استراتيجية الاختبار
| الطبقة | كيف نضمن جودتها |
|--------|-----------------|
| المنطق التربوي النقي (FSRS، اختيار المهارة التالية، تحديث الإتقان، اختيار المستحقات) | **Vitest** — دوال نقية، حالات حدّية، لا عشوائية |
| نداءات AI (المعلّم، المحلّل) | **Zod schemas** + تحقّق + fallback عند عدم المطابقة |
| المنسّق | كود صريح قابل للقراءة، اختبارات تكامل على المسار |
| التيليجرام | اختبار تقسيم الرسائل، الأزرار، معالجة الأخطاء |

**القاعدة:** كل ما يمكن جعله دالة نقية حتمية → يُجعل ويُختبَر. الذكاء (LLM) محصور خلف schemas.

## منظومة التقييم (eval harness — حسم Q3)
كيف **نعرف** أن عمق التصحيح يعمل ولا يتراجع (تفاصيل وأدلة في [09 §Q3](09-open-questions.md)):
- **golden set مؤلّف يدوياً** (150–300 حالة، A1–A2): جملة طالب + التحليل المتوقّع (أخطاء + grammar_points + تصحيح، أو "لا أخطاء"). **النواة يدوية إلزاماً** (GECTurk منشور ⇒ خطر تسرّب). + **25–30% جُمل سليمة** لرصد الإفراط. + عيّنة تنويع من GECTurk movie_reviews (apache-2.0).
- **المقياس: F0.5** (يرجّح الدقّة ×2 — الإفراط أسوأ من التفويت)، **مُنفَّذ بـ TypeScript على نطاقات التحرير** (بلا M2/ERRANT/Python). + **معدّل الإفراط** على الجمل السليمة. لكل مستوى منفصلاً.
- **البوابة (CI):** يُشغَّل عند تغيير prompt/نموذج المعلّم أو المحلّل؛ لا يُشحن تغيير يُنزل F0.5 تحت العتبة أو يرفع الإفراط.
- **الدولاب:** ضغطات 🚩 من الاستخدام الحقيقي (طبقة Q2) تتحوّل لحالات ذهبية جديدة.
- **جودة الشرح:** مُراقَبة بـ LLM-as-judge على عيّنة + تدقيق بشري دوري (لا تُحجَب آلياً).

## جدول المزالق (من النسخ الفاشلة) → الحواجز

| الفخّ الذي أوقع نسخة سابقة | المصدر | الحاجز هنا |
|----------------------------|--------|------------|
| LangGraph 4 عُقد + routing معقّد | TR_AR_AI | منسّق **كود صريح** + دوال نقية، لا graph |
| الاتكاء على Hermes فأفرغ المميزات | TR_AR_AI | محرّكنا يملك المنطق؛ OpenRouter مباشرة |
| تعميق Hermes ← أعطال 502/504 | TR_AR | لا Hermes أصلاً؛ نداء مباشر |
| حالة في الذاكرة تضيع | TR_AR_AI | **Supabase دائماً** |
| بلا streaming / فشل >4096 | TR_AR_AI | تحرير تدريجي + تقسيم |
| بلا logging/معالجة أخطاء | TR_AR_AI | logging مُهيكَل + fallback ودّي |
| اتساع بلا عمق (12 صفحة سطحية) | TR_AR | **عمق أولاً**، نواة صغيرة مُتقنة |
| prompt ضخم بطيء | TR_AR | سياق متدرّج |
| منطق تربوي غير مُختبَر | الكل | Vitest على الدوال النقية |
| انقسام Python/TS | الكل | **TypeScript واحد** |

## مبادئ حاكمة
1. **بساطة صريحة > تجريد ذكي.** كود يُقرأ ويُختبَر، لا أطر تخفي السلوك.
2. **الذكاء في المحادثة، الحتمية في المساندة.** لا تُسلّم LLM ما يمكن أن يكون دالة.
3. **لا فقدان حالة، لا فشل صامت.** كل خطأ مرئيّ في السجل وودّيّ للمستخدم.
4. **عمق قبل اتساع.** لا ميزة جديدة قبل أن تمتاز سابقتها.

## تعريف "يشتغل ممتاز" (قابل للقياس)
- ✅ تصحيح يشرح **السبب النحوي** ويتدرّج بالمستوى (لا "خطأ ← صح").
- ✅ تكيّف ملموس: المعلّم يبني على جلسات سابقة ويتبع مستواك/أهدافك المتغيّرة.
- ✅ محادثة تشعر أنها مع معلّم ذكي (حُكم في اختيار الأنماط).
- ✅ صفر فقدان حالة، صفر فشل صامت، صفر أعطال بوابة.
- ✅ كل منطق تربوي حسّاس مغطّى باختبارات.
