# 08 — الإعداد والمتغيّرات البيئية

> تجميع موحّد لكل ما يحتاجه المشروع للتشغيل — أسرار، نماذج، وإعدادات المعلّم. (القيم الفعلية تُحدَّد في خطة التنفيذ؛ هنا البنية والمتطلّبات.)

## المتغيّرات البيئية المطلوبة (v1)

```
# --- الذكاء الاصطناعي ---
OPENROUTER_API_KEY=...          # المزوّد الرئيسي (نداء مباشر، لا وسيط)
TEACHER_MODEL=...               # نموذج المحادثة (قوي، حُكم تربوي) — قابل للتغيير
ANALYZER_MODEL=...              # نموذج التحليل (سريع، مخرجات منظّمة)

# --- قاعدة البيانات (Supabase) ---
DATABASE_URL=...                # اتصال Drizzle/Postgres
NEXT_PUBLIC_SUPABASE_URL=...
NEXT_PUBLIC_SUPABASE_ANON_KEY=...
SUPABASE_SERVICE_ROLE_KEY=...

# --- التيليجرام ---
TELEGRAM_BOT_TOKEN=...          # @BotFather
TELEGRAM_WEBHOOK_SECRET=...     # التحقق من توقيع webhook
ALLOWED_TELEGRAM_USER_ID=...    # قصر الوصول (تطبيق شخصي)
APP_URL=...                     # لتسجيل الـ webhook (HTTPS)
```

### اختياري (مراحل لاحقة)
```
# المرحلة 2 — الصوت
STT_API_KEY=...                 # تحويل الصوت لنص
TTS_API_KEY=...                 # تحويل النص لصوت + تقييم نطق
# المرحلة 3 — الانغماس / البحث
SERPER_API_KEY=...              # محتوى/بحث ويب (إن لزم)
```

## استراتيجية النماذج
> تقسيم مسنود بالبحث (حسم [Q2](09-open-questions.md)): الأخطاء الصرفية التركية مُوثَّقة حتى للنماذج القوية، فنفصل المحادثة (رخيصة) عن المسار الحرج (قوي).

| المهمة | النموذج | الحرارة | لماذا |
|--------|---------|---------|-------|
| المعلّم (المحادثة الحيّة) | `TEACHER_MODEL` — سريع رخيص | أعلى (~0.7) | حُكم تربوي، طبيعية، تكلفة منخفضة لكل دور |
| **التصحيح + بناء بطاقات SRS** | `ANALYZER_MODEL` — **قوي مُثبَت في التركية** (Claude Sonnet / GPT-4o class) | منخفضة (~0.1) | الدقّة حرجة هنا؛ مخرجات Zod |
| تمريرة مضادة للإفراط (تحقّق) | `ANALYZER_MODEL` | منخفضة | تُشغَّل على التصحيحات فقط (≈2×) |
| تلخيص الجلسة | `ANALYZER_MODEL` | منخفضة | ثبات |

- **نداء مباشر عبر OpenRouter** (لا Hermes، لا subprocess) — تفادياً لهشاشة 502/504.
- فعّل **prompt caching** في OpenRouter (يخفّض 60–80% للسياق المتكرّر).
- النماذج **قابلة للتبديل عبر env** دون تغيير كود. الاختيار النهائي يُحسَم في [Q5](09-open-questions.md).
- ⚠️ تجنّب النماذج المنتهية: `gemini-2.0-flash-001` أُوقف 2026-06-01؛ `gemini-2.5-flash` يُوقَف 2026-10-16.

## التأصيل الحتمي (تبعية إلزامية — حسم Q2)
- **`nlptoolkit-morphologicalanalysis`** (npm، ISC، TypeScript، بلا JVM/Python): محلّل صرفي تركي يعمل داخل Node. يفحص كل كلمة/صيغة تركية مولّدة قبل عرضها (صفر تحليلات = رفض).
- **`nlptoolkit-spellchecker`** (`SimpleSpellChecker` الحتمي): تدقيق إملائي مساند.
- يحدّان من أخطاء الانسجام الصوتي/اللواحق بتكلفة ≈ صفر، دون نداء LLM. (انظر [03 §ضمان صحّة المعلّم](03-pedagogy.md).)

## إعدادات المعلّم (`learner_profile.settings` — jsonb)
مفاتيح تُحيّز سلوك المعلّم (وهو يبقى ذا حُكم، لكنها توجّهه):
- `language_mix`: `arabic_heavy` | `balanced` | `turkish_heavy`
- `correction_strictness`: `gentle` | `moderate` | `strict`
- `response_style`: `casual` | `formal`
- `focus_areas`: قائمة (نحو، مفردات، محادثة، نطق…)
- `custom_instructions`: نص حر من المستخدم

> ملاحظة: لا نُعيد نظام "موافقة التيليجرام على تغيير الإعدادات" من النسخة القديمة — خارج نطاق v1 (تبسيط مقصود).

## إدارة الأسرار
- محليّاً: `.env` (مُستبعَد من git عبر `.gitignore`).
- إنتاجاً: متغيّرات بيئة الحاوية/الخادم. لا أسرار في الكود أو الـ repo.
